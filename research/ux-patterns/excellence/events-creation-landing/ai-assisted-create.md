# Pattern: AI-assisted creation (structured Q&A drafts the page; AI as named helper, never the only path)

**Surface:** events-creation-landing · **Observed in:** Eventbrite web (https://mobbin.com/screens/b61d5937-ec5d-4ff3-9341-bdbce3bb6a7a, https://mobbin.com/flows/f24e6aaf-d4d3-479f-94de-586fb488c4ee, https://mobbin.com/flows/595fc812-4448-484b-aee7-4cf12a7760c0), Partiful iOS (https://mobbin.com/flows/732d3782-570a-47cc-9a84-cd991745c03c), Luma web (https://mobbin.com/flows/503f562f-e287-4127-99b0-a38cdaa5fd6c), Notion web — analogous (https://mobbin.com/screens/c562f7e8-d22a-45ac-b255-b73d72b34d34)

## Flow
1. Entry is a fork, never a default: "Start from scratch — Add all your event details…" vs "✨ Create my event faster with AI — Answer a few quick questions to generate an event that's ready to publish almost instantly"; the AI page repeats the exit: "You can still create an event without AI." (Eventbrite).
2. AI input is STRUCTURED Q&A, not a chatbot: name (helper: "Your title will be used to help create your event's summary, description, category, and tags – so be specific!"), dates, location mode, price, capacity → "Create event" → loading: "Thanks for giving us a spin! Give us a moment to build your event…" — AI fills downstream fields (summary, description, tags), the organizer edits in the normal builder after.
3. Capability limits stated honestly inline: "Our tool can only generate one General Admission ticket for now. You can edit and add more ticket types later."
4. Micro-assists inside fields beat wizard AI: "Suggest Description" link in the corner of the description field (Luma).
5. AI personified for ideation: "🧞 Need inspo? Ask the Party Genie", vibe chips + "Randomize" + "✨ Or create from scratch" (Partiful).
6. Generated content always carries a review disclaimer: "This caption was generated by Chat GPT … Remember to review and edit your content for accuracy before sharing." + "Generate another caption" (Eventbrite social composer).

## Use when
Organizers face blank-page content work (descriptions, summaries). Field-level assists ("Suggest Description") are the highest value-to-effort variant.

## Avoid when
Never make AI the only path or auto-publish its output; medical/academic content demands review (accuracy disclaimers are observed practice even for party captions). Skip personified AI for professional tools.

## Sad paths observed
- Capability limits disclosed at point of use (verbatim above) rather than failing after.
- Regeneration affordance ("Generate another caption") assumes the first draft disappoints — design for it.

## Accessibility
AI entry is a labeled card with a text alternative path; loading states are text sentences.

## Default verdict for our stack
VIABLE (V2) — V1 ships field-level "Suggest description" at most. The structured-Q&A generator is a strong V2 once the page-section model exists to fill. Always with review-before-publish and an opt-out path.
