# Lazy Dev — Multi-Phase Improvement Orchestrator

Chains multiple `/improve` loops in a safety-ordered sequence. Drives all underperforming metrics toward their targets in one session or generates an overnight script.

Adapted from lazy-developer's phase orchestration pattern.

## Usage

```
/lazy-dev
/lazy-dev safety-first
/lazy-dev speed-first --intensity aggressive
/lazy-dev custom --overnight
```

## Presets

### safety-first (default)

Fix safety issues first, then coverage, then performance:

```
1. tsc-errors   → target 100  (lock tests, edit source)
2. eslint       → target 100  (lock tests, edit source)
3. coverage     → target 80   (lock source, edit tests)
4. mutation     → target 75   (lock source, edit tests)
5. axe-violations → target 100 (lock tests, edit source)
6. lighthouse-perf → target 90 (lock tests, edit source)
7. lighthouse-a11y → target 95 (lock tests, edit source)
```

### speed-first

Fix test quality first to establish trust, then source:

```
1. coverage     → target 80   (lock source, edit tests)
2. mutation     → target 75   (lock source, edit tests)
3. tsc-errors   → target 100  (lock tests, edit source)
4. eslint       → target 100  (lock tests, edit source)
5. axe-violations → target 100 (lock tests, edit source)
6. lighthouse-perf → target 90 (lock tests, edit source)
```

### custom

User specifies the order and targets.

## Workflow

### 1. Configuration

Ask the user (or accept as arguments):
- **Preset**: safety-first / speed-first / custom
- **Execution mode**: interactive (run now) / overnight (generate script)
- **Intensity**: quick (5 iters) / moderate (10 iters) / aggressive (20 iters)
- **Stopping strategy**: targets-met / diminishing-returns / max-time

### 2. Discovery — Establish baselines

Run all metrics and record current scores:

```bash
for metric in tsc-errors eslint coverage mutation axe-violations lighthouse-perf lighthouse-a11y; do
  ./scripts/score.sh "$metric"
done
```

### 3. Initialize state

Create `.quality/goals/lazy-dev-state.md`:

```markdown
# Lazy Dev Pipeline State

Started: <ISO timestamp>
Preset: <preset>
Intensity: <level>
Mode: <interactive | overnight>

## Pipeline

| # | Metric | Baseline | Target | Status | Final | Delta |
|---|--------|----------|--------|--------|-------|-------|
| 1 | tsc-errors | 85 | 100 | pending | - | - |
| 2 | eslint | 72 | 100 | pending | - | - |
| 3 | coverage | 65 | 80 | pending | - | - |
| ... | ... | ... | ... | ... | ... | ... |
```

### 4. Execute phases (interactive mode)

For each phase in order:

1. **Skip check**: If current score >= target, mark "skipped" and advance
2. **Pre-flight**: Verify build passes and tests pass
3. **Run `/improve`**: Execute the improvement loop for this metric with the configured intensity
4. **Record result**: Update the state table with final score and delta
5. **Regression check**: Run ALL previous metrics to verify nothing regressed
6. **Advance**: Move to next phase

Between phases, announce progress:
```
Phase 2/7 complete: eslint 72 → 95 (+23). Moving to coverage.
```

### 5. Execute phases (overnight mode)

Instead of running interactively, generate `.quality/goals/overnight-improve.sh`:

```bash
#!/bin/bash
# Generated by /lazy-dev on <date>
# Preset: safety-first, Intensity: moderate

set -e
MAX_TURNS=10

METRICS=("tsc-errors" "eslint" "coverage" "mutation" "axe-violations" "lighthouse-perf" "lighthouse-a11y")
TARGETS=(100 100 80 75 100 90 95)

for i in "${!METRICS[@]}"; do
  METRIC="${METRICS[$i]}"
  TARGET="${TARGETS[$i]}"
  CURRENT=$(./scripts/score.sh "$METRIC" | grep -o '"score": [0-9]*' | grep -o '[0-9]*')

  if [ "$CURRENT" -ge "$TARGET" ]; then
    echo "SKIP: $METRIC already at $CURRENT (target: $TARGET)"
    continue
  fi

  echo "IMPROVE: $METRIC from $CURRENT toward $TARGET"

  claude -p "/improve $METRIC --target $TARGET --intensity moderate" \
    --allowedTools 'Edit,Write,Bash(git *),Bash(./scripts/score.sh *),Bash(npx *)' \
    2>&1 | tee ".quality/goals/improve-$METRIC.log"

  FINAL=$(./scripts/score.sh "$METRIC" | grep -o '"score": [0-9]*' | grep -o '[0-9]*')
  echo "RESULT: $METRIC $CURRENT → $FINAL"
done

echo "Overnight improvement complete. Run full QA harness for regression check."
```

Make it executable: `chmod +x .quality/goals/overnight-improve.sh`

## File-Locking Rules

| Phase Metrics | Agent May Edit | Agent Must NOT Edit |
|---------------|---------------|-------------------|
| tsc-errors, eslint | `src/`, `app/`, `lib/`, `components/` | `*.test.*`, `*.spec.*`, `__tests__/`, `tests/` |
| coverage, mutation | `*.test.*`, `*.spec.*`, `__tests__/`, `tests/` | `src/`, `app/`, `lib/`, `components/` |
| axe, lighthouse-*, playwright | `src/`, `app/`, `lib/`, `components/` | `*.test.*`, `*.spec.*`, `__tests__/`, `tests/` |

These locks prevent metric gaming (e.g., deleting tests to improve mutation score).

## Safety Constraints

- Tier 1 modules are NEVER included in overnight mode
- Every phase verifies previous metrics haven't regressed
- If ANY metric drops below its starting baseline, the pipeline halts
- The QA harness must run after completion for independent verification
